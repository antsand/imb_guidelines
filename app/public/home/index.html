Application development with User Stories in mind
Who is this guide for?
Everyone involved in creating an application
  • Product Owners
  • Scrum Master
  • Devops
  • Full Stack developers
Why this guide and what is the current problem?
To diagnose the current issues, let us focus on the fronend of the application
A Monolith contains all the components under a single system. One team works on the whole system.
Microservices modularize different components of a system. Multiple teams can work on the system.
Though the system can be micro-serivce the frontend in most cases is monolithic.
Approximately 80% of our code base is frontend code
80% of our micro-service architecture only focuses on 20% of our code base
An advanced microservice frontend would require each user story to have it's own container
How do we know when to use basic vs advanced micro-frontend?
  • How different are the users for each user stories?
  • Should different teams work on each user story?
  • What is the expected traffic for the application?
  • How much bandwidth (e.g. data) does each user story require?
  • How complex is each user story, in terms of code complexity?
  • How tightly coupled are each user story
